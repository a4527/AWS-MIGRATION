# AGENTS.md

이 저장소에서 작업할 때의 운영 규칙을 정리합니다.

## 기본 원칙

- 문서는 `docs/`에 목적별로 분리한다.
- 구현 코드는 `backend/`, `terraform/`, `kubernetes/`에 역할별로 배치한다.
- 온프레미스와 AWS 구현은 문서상 명확히 구분한다.
- 마이그레이션 관련 변경은 반드시 `docs/05_MIGRATION.md`와 함께 갱신한다.

## 문서 작성 기준

- 기능 설명은 사용자 관점과 운영 관점을 함께 담는다.
- API 문서는 요청/응답 예시와 에러 코드를 포함한다.
- 아키텍처 문서는 데이터 흐름과 구성요소 책임을 분리해서 적는다.
- 배포 문서는 로컬 실행과 운영 배포를 분리해서 설명한다.

## 변경 시 우선순위

1. 요구사항
2. 아키텍처
3. 데이터 모델
4. API
5. 배포
6. 운영/트러블슈팅
